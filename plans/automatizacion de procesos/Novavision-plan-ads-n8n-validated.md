## CONTEXTO DE MERCADO — POR QUÉ FUNCIONA ESTO AHORA

### El momento de NovaVision en números reales (fuentes verificadas)

El e-commerce argentino creció **60% interanual en 2025** (CACE), superando la inflación del 31.5% por 28.5 puntos — o sea, crecimiento real del volumen de ventas. El 18% del retail argentino ya es online, casi el doble del promedio regional (10%). Las marcas de Tiendanube que operaron en 2024 y 2025 crecieron un **40.2% real neto**.

El dolor concreto que NovaVision resuelve, validado por datos:
- El **87.5% de las ventas por redes sociales** en Argentina van por Instagram (NubeCommerce 2025)
- El **97.25% de las marcas** que venden online usan Instagram para difundir — pero sin tienda propia, la gestión es completamente manual
- El **56% de emprendedores** vende solo online, sin local físico — necesitan una solución de e-commerce seria
- El **56.3% de emprendedores** ya usa IA para crear contenido — el mercado está listo para novedades tecnológicas
- La adquisición de clientes es el **mayor reto declarado** del sector (CACE 2025)

## MENSAJES DE VENTA VALIDADOS CONTRA LOS DOLORES REALES

### El ICP en detalle (basado en datos de mercado)

**Perfil primario:**
- Emprendedor/a 25-44 años (67% mujeres según Tiendanube)
- Vende por Instagram y/o WhatsApp hace 6-36 meses
- Tiene entre 500 y 5.000 seguidores en Instagram con engagement real
- Proceso de venta: DM → acuerda precio → pide transferencia → coordina envío manualmente
- Problema: pierde tiempo, pierde ventas porque "no tiene tienda", no puede escalar

**Perfil secundario:**
- Negocio físico (ropa, artesanías, comida, accesorios) que quiere canal online
- Ya usa Mercado Libre pero le duelen las comisiones y las reglas del marketplace
- Quiere "independencia" de la plataforma

### Los 5 ángulos con mayor potencial de conversión (priorizados por validación)

```
ÁNGULO 1 — "El tiempo que perdés" (mayor dolor declarado)
Hook: "Respondés pedidos por WhatsApp durante 2 horas al día. 
       En un año, eso son 30 días de tu vida."
Mensaje: NovaVision automatiza los pedidos. Tu tienda vende mientras dormís.
Por qué funciona: La "pérdida de tiempo en DMs" es el dolor #1 de vendedores 
de Instagram. Dato CACE: la gestión manual es el mayor freno al crecimiento.

ÁNGULO 2 — "0% de comisión" (diferenciador financiero inmediato)
Hook: "MercadoShops se queda con una parte de cada venta. Nosotros, no."
Mensaje: Pagás un fee fijo mensual. Todo lo que vendés es tuyo.
Por qué funciona: Con e-commerce creciendo 60%, las comisiones de marketplaces 
se sienten cada vez más. Argumento financiero concreto y medible.

ÁNGULO 3 — "Tienda en minutos" (velocidad, sin fricción técnica)
Hook: "Creé mi tienda online sin saber nada de diseño ni programación."
Formato: Screen recording real del proceso de onboarding (vos mismo grabándolo)
Por qué funciona: La percepción de "dificultad técnica" es la barrera 
de entrada más declarada por emprendedores para no tener tienda propia.

ÁNGULO 4 — "Mercado Pago + envíos nativos" (reducción de fricción operativa)
Hook: "Tu cliente paga con Mercado Pago. El paquete sale con Andreani.
       Vos no hacés nada manual."
Por qué funciona: Mercado Pago tiene penetración masiva en Argentina.
Andreani/OCA son los operadores que los emprendedores ya conocen.
La combinación elimina las dos principales fricciones post-venta.

ÁNGULO 5 — "SEO con IA" (diferenciador tecnológico)
Hook: "Mi tienda aparece en Google sin que yo haga nada.
       La IA se encarga sola."
Por qué funciona: 56.3% de emprendedores ya usa IA para contenido.
El mercado entiende el valor de IA aplicada al negocio.
Es un diferenciador que Tiendanube y MercadoShops no tienen nativamente.
```

### Mensajes por etapa del funnel

```
TOP OF FUNNEL (awareness — no saben que necesitan esto):
→ "¿Cuánto tiempo perdés respondiendo pedidos por Instagram?"
→ "Vendés por WhatsApp pero ¿tenés control de tu stock?"
→ "¿Sabías que podés tener tu propia tienda con Mercado Pago sin pagar comisión?"

MIDDLE OF FUNNEL (consideración — comparan opciones):
→ "NovaVision vs. Tiendanube vs. vender por redes: calculá el costo real"
→ "Setup en 5 minutos. 20 tutoriales interactivos. Sin llamada de ventas."
→ Mostrar el panel admin — la tienda se ve profesional y real

BOTTOM OF FUNNEL (decisión — listos para pagar):
→ Cupón FUNDADORES2026: primeras 50 tiendas con 40% off Growth mes 1
→ "Probalo 14 días. Si no te convence, te devolvemos el dinero."
→ Testimonial de primera tienda activa (urbanprint u otro real)
```

## ESPECIFICACIONES COMPLETAS DE WORKFLOWS n8n

### Prerequisitos de credenciales (cargar en n8n antes de cualquier workflow)

```
En n8n → Settings → Credentials, crear:

1. META_MARKETING_API
   Tipo: HTTP Header Auth
   Nombre: Meta Marketing API
   Header: Authorization
   Value: Bearer {ACCESS_TOKEN}
   
   Cómo obtener el access token:
   a. developers.facebook.com → Create App → Business
   b. Add product: Marketing API
   c. Generate System User token (no personal, no expira)
   d. Permisos necesarios: ads_management, ads_read, business_management,
      pages_read_engagement, instagram_basic

2. GOOGLE_ADS_OAUTH
   Tipo: Google Ads (nodo nativo n8n)
   Client ID + Client Secret desde Google Cloud Console
   Developer Token: solicitarlo en Google Ads → Tools → API Center
   Manager Account ID: tu MCC si usás uno

3. TIKTOK_ADS_API
   Tipo: HTTP Header Auth
   Header: Access-Token
   Value: {TIKTOK_ACCESS_TOKEN}
   
   Cómo obtener:
   a. business.tiktok.com → Developer → Create App
   b. Scopes: ad.read, report.read, campaign.read
   c. Token type: Long-lived (no expira en 24hs)

4. OPENAI_API
   Tipo: HTTP Header Auth
   Header: Authorization
   Value: Bearer {OPENAI_API_KEY}

5. SUPABASE_ADMIN
   Tipo: HTTP Header Auth
   Header: apikey
   Value: {SUPABASE_ADMIN_SERVICE_ROLE_KEY}
   
6. WHATSAPP_BUSINESS
   Tipo: HTTP Header Auth  
   Header: Authorization
   Value: Bearer {WHATSAPP_ACCESS_TOKEN}
```

## SQL FUNCTIONS NECESARIAS EN SUPABASE

Ejecutar en el SQL Editor de Supabase Admin DB:

```sql
-- Función 1: Métricas diarias de funnel
CREATE OR REPLACE FUNCTION get_daily_funnel_metrics(p_date DATE)
RETURNS JSON AS $$
DECLARE
  result JSON;
BEGIN
  SELECT json_build_object(
    'registrations', (
      SELECT COUNT(*) FROM nv_accounts 
      WHERE DATE(created_at) = p_date
    ),
    'onboardings_started', (
      SELECT COUNT(*) FROM nv_onboarding 
      WHERE DATE(created_at) = p_date
    ),
    'onboardings_completed', (
      SELECT COUNT(*) FROM nv_onboarding 
      WHERE DATE(updated_at) = p_date 
      AND status IN ('provisioned', 'live', 'approved')
    ),
    'payments', (
      SELECT COUNT(*) FROM subscriptions 
      WHERE DATE(created_at) = p_date 
      AND status = 'active'
    ),
    'current_mrr', (
      SELECT COALESCE(SUM(
        CASE plan_key 
          WHEN 'starter' THEN 20
          WHEN 'growth' THEN 60
          WHEN 'enterprise' THEN 390
          WHEN 'starter_annual' THEN (200.0/12)
          WHEN 'growth_annual' THEN (600.0/12)
          WHEN 'enterprise_annual' THEN (3500.0/12)
          ELSE 0 
        END
      ), 0) 
      FROM subscriptions s
      JOIN nv_accounts a ON a.id = s.account_id
      WHERE s.status = 'active'
    )
  ) INTO result;
  RETURN result;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

-- Función 2: Métricas semanales de funnel
CREATE OR REPLACE FUNCTION get_weekly_funnel_metrics(p_days INTEGER DEFAULT 7)
RETURNS JSON AS $$
BEGIN
  RETURN (
    SELECT json_build_object(
      'registrations', COUNT(DISTINCT a.id),
      'registrations_prev_week', (
        SELECT COUNT(*) FROM nv_accounts 
        WHERE created_at >= NOW() - INTERVAL '14 days'
        AND created_at < NOW() - INTERVAL '7 days'
      ),
      'onboardings_completed', (
        SELECT COUNT(*) FROM nv_onboarding 
        WHERE updated_at >= NOW() - INTERVAL '7 days'
        AND status IN ('provisioned', 'live', 'approved')
      ),
      'new_payments', (
        SELECT COUNT(*) FROM subscriptions 
        WHERE created_at >= NOW() - INTERVAL '7 days'
        AND status = 'active'
      ),
      'current_mrr', (
        SELECT COALESCE(SUM(
          CASE plan_key 
            WHEN 'starter' THEN 20 WHEN 'growth' THEN 60 WHEN 'enterprise' THEN 390
            WHEN 'starter_annual' THEN 16.67 WHEN 'growth_annual' THEN 50
            WHEN 'enterprise_annual' THEN 291.67 ELSE 0 
          END
        ), 0) FROM subscriptions WHERE status = 'active'
      ),
      'outreach_contacts_7d', (
        SELECT COUNT(DISTINCT lead_id) FROM outreach_logs 
        WHERE created_at >= NOW() - INTERVAL '7 days'
        AND type IN ('WA_SENT', 'EMAIL_SENT')
      ),
      'outreach_responses_7d', (
        SELECT COUNT(DISTINCT lead_id) FROM outreach_logs 
        WHERE created_at >= NOW() - INTERVAL '7 days'
        AND type = 'REPLY_RECEIVED'
      ),
      'hot_leads', (
        SELECT COUNT(*) FROM outreach_leads 
        WHERE engagement_score >= 80 
        AND status = 'IN_CONVERSATION'
      )
    )
    FROM nv_accounts a
    WHERE a.created_at >= NOW() - INTERVAL '7 days'
  );
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```
