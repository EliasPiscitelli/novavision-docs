## Resumen ejecutivo

NovaVision tiene una **infraestructura IA técnicamente madura** (credit system universal, key pool, Store DNA, 10 action codes, guards, pricing dinámico) pero con **adopción real cercana a cero**. Solo se consumieron ~75 créditos IA en total (mayormente de testing), 1 solo tenant tiene Store DNA generado, y los campos de personalización de marca están vacíos en todas las cuentas. El modelo de créditos universales está correctamente implementado (3 packs activos, los per-feature se desactivaron a propósito). La `ANTHROPIC_API_KEY` está configurada pero sin uso en código. SEO AI tiene su propio sistema de créditos separado (por diseño). Los 5 workflows n8n con IA están inactivos. **Falta crítica**: no se validan los límites operativos del tenant (max productos, FAQs, banners) antes de permitir generación IA. El sistema está construido pero no encendido.

### B. Código fuente

#### Módulo `ai-generation/` (API)
- **10 action codes** implementados con controller + service + prompts + guard
- **Patrón consistente**: Guard valida créditos → Service genera con OpenAI → Controller consume créditos solo si exitoso
- **Retry**: 2-3 intentos con backoff exponencial (2-5s)
- **Store DNA**: Se inyecta como prefijo del system prompt en todas las generaciones
- **Todos los prompts en español rioplatense** — correcto para el mercado AR
- **Response format**: JSON estricto (`response_format: { type: 'json_object' }`)

#### Módulo `ai-credits/` (API)
- **Pool universal**: Migrado de créditos per-feature a pool universal (evidencia: welcome configs per-feature inactivas, solo `ai_universal` activo)
- **Guard 402**: Retorna error estructurado con `required`, `available`, `addon_store_url`
- **Reserve/Refund**: Para jobs async (catalog generation)
- **Promo credits**: Endpoint admin para grants manuales

#### Módulo `seo-ai/` (API)
- **Worker polling**: 10s interval, chunks de 25, max 1 job/tenant, 5 jobs/día
- **Modelo**: gpt-4o-mini, T=0.3, max_tokens=500
- **Ledger separado**: Usa `seo_ai_credits` (vacía) en vez del `account_action_credit_ledger`
- **HALLAZGO**: SEO AI y AI Credits son **dos sistemas de créditos paralelos** que no están unificados

#### Módulo `marketing/` (API)
- **3 servicios**: Creative Studio, Campaign Advisor, Audience Intel
- **Modelo**: gpt-4o (creative, T=0.4-0.7)
- **Acceso**: Super-admin only — NO tenant-facing
- **Integración n8n**: Webhook para publicar creativos
- **Fallback**: Rule-based si OpenAI no disponible

#### Módulo `support/` (API)
- **SupportAiService**: Genera prompts para Claude Code (human-in-the-loop)
- **NO hace llamadas LLM**: Solo construye contexto + prompt para que un humano copie y ejecute
- **Pipeline**: open → dev → qa → done con approval workflow
- **Tabla `ticket_tasks`**: Planificada pero no verificada si fue creada

#### Frontend Admin
- **AiAnalysisSection**: Componente en SupportConsoleView para generar/copiar prompts Claude
- **AiCreditsPricingView**: Dashboard super-admin 4 tabs (pricing, packs, welcome, client credits)
- **NovaVisionChatAgent**: Widget iframe a n8n webhook (chat conversacional)

#### Frontend Web (Storefront/Tenant Dashboard)
- **Botones IA** en: ProductModal, QADashboard, ReviewsDashboard, FaqSection, BannerSection, ImportWizard, LogoSection, ServiceSection
- **AiTierToggle**: Toggle Normal/Pro en cada botón IA
- **Interceptor 402**: Global para créditos insuficientes → modal → redirect a Addon Store
- **Tours**: Steps de IA documentados en products, banners, FAQs
